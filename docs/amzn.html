Introduccin

Antecedentes

El anlisis de activos financieros es bastante popular e importante en finanzas y las series de tiempo son ampliamente utilizadas para estudiar estos procesos financieros. Los datos usados en este ejercicio corresponden al precio diario de AMAZON (AMZN) para el periodo 2019-10-21 y 2020-10-20, que se pueden ser extrados del sitio web de Yahoo Finance. El conjunto de datos contiene los precios de cierre, cierre ajustado, de apertura, ms alto, y ms bajo. Ac se usa el precio ajustado de cierre como medida general del precio de AMAZON.

Objetivo

Construir el modelo adecuado para AMZN. Extraer el componente aleatorio, eliminando el componente estacional y el componente de tendencia, y utilizar el logaritmo de la serie para estabilizar los datos. Posteriormente, se ajusta el modelo ARMA a los datos, seleccionando los valores \(p\) y \(q\). Finalmente, a partir del modelo estimado se realiza una proyeccin y se evalua la calidad de la misma.

A partir de la ACF, es posible evidenciar autocorrelacin (aunque pequea) en los rezagos 8, 22, 23 y 33, se debe descomponer la serie para obtener datos estacionarios.

Las series de tiempo pueden ser descompuestas en tres componentes: tendencia, componente estacional y componente aleatorio, los cuales se denotan como \(T_t\), \(S_t\) y \(R_t\) respectivamente, denotando la serie de tiempo original como \(Y_t\), es posible escribir \(Y_t\) como una funcin de los tres componentes:

\[Y_t = f(T_t, S_t, R_t)\]

Para la transformacin logartmica, la funcin \(f\) toma la forma aditiva:

\[Y_t = T_t+S_t+R_t\]

El objetivo de identificar los diferentes componentes es separar el componente aleatorio de los otros componentes de la serie. Esto es, se quiere eliminar el ruido y aislar la seal. El siguiente grfico muestra la descompsicin de los datos de AMZN.

amz_ts <- ts(amz_df,frequency = 365,start = 2010-01-01 )
amz_de <- decompose(amz_ts)

Para estimar el modelo \(ARMA(p,q)\) se deben seleccionar \(p\) y \(q\) utilizando el criterio AIC.

amz_rand <- amz_de$random
aic_table <- function(data,P,Q){
  table <- matrix(NA,(P+1),(Q+1))
  for(p in 0:P) {
    for(q in 0:Q) {
       table[p+1,q+1] <- arima(data,order=c(p,0,q))$aic
    }
  }
  dimnames(table) <- list(paste("AR",0:P, sep=""),paste("MA",0:Q,sep=""))
  table
}

Pronstico

El modelo estimado se utiliza para pronsticar el retorno de AMAZON. Para tal fin, los datos se entrenan con el 90% de la muestra y con el otro 10% se realizan los pronsticos y se mide la precisin usando el criterio \(RSME\)

amz_train <- amz_rand[1:((0.9)*length(amz_rand))]
amz_test <- amz_rand[(0.9*length(amz_rand)):length(amz_rand)]
train12 <- arima(amz_train, order = c(1,0,2))
pred <- predict(train12, n.ahead = (length(amz_rand)-(0.9*length(amz_rand))))$pred
forecast <- forecast(train12, h = 50)
